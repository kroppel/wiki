SSH

Creating an ssh configuration file

An ssh configuration file can make it easier for you to connect to servers that you access frequently. In one case it also helped me with the VS Code SSH Remote Access Plugin. ssh was not able to find my private ssh key for authentication, so I created an ssh config file in which i specified the key that ssh should use for this connection.

Creating an ssh configuration file is just as easy as it sounds. The common place for the config file to reside is under .ssh inside your home directory. Create the file for example with the command touch config. Here's an example of a config file's content:


              Host <name>
                HostName <ip-address>
                User <username>
                IdentityFile <path>

  • <name> specifies a name that can be referenced to quickly establish the configured connection, e.g. via ssh <name>
  • <ip-address> specifies the ip adress of the server that you want to connect to
  • <username> specifies as which user the connection should be established
  • <path> specifies the path to the private ssh key that should be used for authentication

Git

Github authentication via access token

A github access token can be created under "Settings" -> "Developer settings" -> "Personal access token".

To execute a push, simply use the following template: git push https://<GITHUB_ACCESS_TOKEN>@github.com/<GITHUB_USERNAME>/<REPOSITORY_NAME>.git

BIOS

Booting from an USB device

Booting from an USB device is helpful for setting up a new OS installation, or if you want to keep your whole FS portable on the USB stick itself. Assuming that you already created a bootable USB stick, e.g. with https://www.balena.io/etcher/, the usual workflow is to enter the BIOS setup on startup and to change the boot order so that the system primarily tries to boot from your USB.

On older systems you might have a hard time finding the USB boot option. In this case it's always helpful to update your BIOS to the newest version available for your motherboard. These can usually be found on the manufacturer's website. This problem occured on my Acer-Aspire-7750-G, on which I wanted to install Linux Mint XFCE. A bios update solved the issue.
